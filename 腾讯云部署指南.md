# 服务器部署指南（使用本地MySQL）

## 📋 准备工作

### 1. 购买服务器
- 推荐配置：2核4G或更高
- 操作系统：Ubuntu 20.04 LTS 或 CentOS 7+
- 带宽：10Mbps或更高

### 2. 数据库选择
- **推荐方案**：使用服务器本地MySQL（简单、快速、配置少）
- **替代方案**：使用腾讯云MySQL（需要配置内网连接，较复杂）

**当前使用：本地MySQL** ✅

---

## 🚀 部署步骤

### 第一步：初始化服务器

#### SSH连接服务器
```bash
ssh root@<你的服务器IP>
```

#### 更新系统软件包
```bash
# Ubuntu/Debian
sudo apt update && sudo apt upgrade -y

# CentOS/RHEL
sudo yum update -y
```

#### 安装必要工具
```bash
# Ubuntu/Debian
sudo apt install -y curl wget git vim unzip

# CentOS/RHEL
sudo yum install -y curl wget git vim unzip
```

---

### 第二步：安装本地MySQL数据库

#### Ubuntu/Debian系统
```bash
# 安装MySQL服务器
sudo apt install -y mysql-server

# 启动MySQL服务
sudo systemctl start mysql
sudo systemctl enable mysql

# 检查状态
sudo systemctl status mysql

# 配置MySQL安全
sudo mysql_secure_installation
# 按提示设置root密码（建议：设置强密码）
# Y - 移除匿名用户
# Y - 禁止root远程登录
# Y - 删除test数据库
# Y - 重新加载权限表
```

#### CentOS/RHEL系统
```bash
# 安装MySQL
sudo yum install -y mysql-server

# 启动MySQL
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 设置root密码
sudo mysql -e "ALTER USER 'root'@'localhost' IDENTIFIED BY '你的强密码';"
```

---

### 第三步：创建数据库

#### 方法一：使用命令行
```bash
# 登录MySQL
mysql -u root -p

# 执行以下SQL
CREATE DATABASE website_batch_system CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 授权
GRANT ALL PRIVILEGES ON website_batch_system.* TO 'root'@'localhost';
FLUSH PRIVILEGES;

# 验证
SHOW DATABASES;
# 应该能看到 website_batch_system

# 退出
EXIT;
```

#### 方法二：使用SQL脚本
```bash
# 创建SQL文件
nano create_database.sql
```

写入以下内容：
```sql
CREATE DATABASE IF NOT EXISTS website_batch_system
    CHARACTER SET utf8mb4
    COLLATE utf8mb4_unicode_ci;

GRANT ALL PRIVILEGES ON website_batch_system.* TO 'root'@'localhost';
FLUSH PRIVILEGES;
```

```bash
# 执行脚本
mysql -u root -p < create_database.sql

# 验证
mysql -u root -p -e "SHOW DATABASES;"
```

### 第三步：安装Java环境

#### 安装JDK 17
```bash
# 使用SDKMAN安装（推荐）
curl -s "https://get.sdkman.io" | bash

sdk install java 17.0.8-tem
```

#### 或使用APT安装（Ubuntu/Debian）
```bash
sudo apt install -y openjdk-17-jdk
```

#### 配置JAVA_HOME
```bash
echo 'export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64' >> ~/.bashrc
echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

### 第四步：部署后端服务

#### 1. 上传项目代码到服务器

**方法1：使用Git**
```bash
git clone <你的仓库地址> /opt/website-batch-system
cd /opt/website-batch-system/backend
```

**方法2：使用SCP上传**
```bash
# 在本地打包
cd backend
tar -czf backend.tar.gz .

# 上传到服务器
scp backend.tar.gz root@<服务器IP>:/opt/

# 在服务器上解压
ssh root@<服务器IP>
cd /opt
tar -xzf backend.tar.gz
```

#### 2. 修改数据库配置

编辑配置文件：
```bash
nano /opt/website-batch-system/backend/src/main/resources/application-cloud.yml
```

修改以下配置：
```yaml
spring:
  datasource:
    url: jdbc:mysql://cdb-xxx.sql.tencentcdb.com:3306/website_batch_system?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: <你的数据库用户名>
    password: <你的数据库密码>
```

#### 3. 编译和打包
```bash
cd /opt/website-batch-system/backend

# 编译打包
mvn clean package -DskipTests

# 打包后的jar文件在 target/ 目录下
```

#### 4. 创建系统服务（使用systemd）

创建服务文件：
```bash
sudo nano /etc/systemd/system/website-batch-system.service
```

写入以下内容：
```ini
[Unit]
Description=Website Batch System Backend
After=network.target mysql.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/website-batch-system/backend
ExecStart=/usr/lib/jvm/java-17-openjdk-amd64/bin/java -jar target/website-batch-system-1.0.0.jar --spring.profiles.active=cloud
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
# 重新加载systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start website-batch-system

# 设置开机自启
sudo systemctl enable website-batch-system

# 查看服务状态
sudo systemctl status website-batch-system
```

#### 5. 检查后端服务

访问：http://<服务器IP>:8080/api/excel/companies

应该看到返回的JSON数据（可能是空数组）。

### 第五步：部署前端静态文件

#### 1. 构建前端

在本地机器上构建：
```bash
cd frontend
npm install
npm run build
```

生成的文件在 `frontend/dist` 目录下。

#### 2. 上传到服务器

**方法1：使用SCP上传**
```bash
# 打包dist目录
cd frontend
tar -czf dist.tar.gz dist/

# 上传到服务器
scp dist.tar.gz root@<服务器IP>:/opt/
```

**方法2：使用宝塔面板**
1. 在宝塔面板中，进入"文件"管理
2. 上传 dist.tar.gz 到 /opt/ 目录
3. 解压到 /var/www/html/ 目录

#### 3. 部署到Nginx

创建或编辑Nginx配置文件：
```bash
sudo nano /etc/nginx/sites-available/website-batch-system
```

写入以下内容：
```nginx
server {
    listen 80;
    server_name <你的域名或IP>;

    # 后端API代理
    location /api/ {
        proxy_pass http://localhost:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 前端静态文件
    location / {
        root /var/www/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 日志
    access_log /var/log/nginx/website-batch-system-access.log;
    error_log /var/log/nginx/website-batch-system-error.log;
}
```

启用配置：
```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/website-batch-system /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载Nginx
sudo systemctl reload nginx
```

### 第六步：配置域名（可选）

#### 1. 购买域名

在腾讯云购买域名。

#### 2. 添加域名解析

在腾讯云DNS解析中添加：
- 记录类型：A
- 主机记录：@
- 记录值：你的服务器IP
- TTL：600

#### 3. 配置SSL证书（推荐）

使用Let's Encrypt免费证书：

```bash
# 安装certbot
sudo apt install -y certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d <你的域名>

# 自动续期
sudo certbot renew --dry-run
```

Nginx配置会自动添加HTTPS配置。

## 📊 防火墙配置

### 腾讯云安全组
在腾讯云控制台配置安全组规则：
- 开放端口：
  - 80（HTTP）
  - 443（HTTPS）
  - 8080（后端API，如果需要直接访问）

### 服务器防火墙（UFW）
```bash
# 安装UFW
sudo apt install -y ufw

# 开放端口
sudo ufw allow 80
sudo ufw allow 443

# 启用防火墙
sudo ufw enable

# 查看状态
sudo ufw status
```

## 🐛 故障排查

### 1. 后端无法启动

检查日志：
```bash
sudo journalctl -u website-batch-system -n 100
```

常见问题：
- 数据库连接失败：检查数据库IP、端口、用户名、密码
- 端口被占用：检查8080端口是否被其他程序占用
- 依赖缺失：重新运行 `mvn clean package`

### 2. 前端无法访问

检查：
- Nginx是否正常运行：`sudo systemctl status nginx`
- 文件权限：检查 `/var/www/html` 目录权限
- Nginx配置是否正确：`sudo nginx -t`

### 3. API调用失败

- 检查后端服务是否启动
- 检查Nginx代理配置
- 检查跨域配置（CORS）
