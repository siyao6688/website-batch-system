# 项目更新部署指南

本文档描述如何将本地修改的代码部署到腾讯云服务器。

## 📋 服务器信息

- **服务器IP**: `124.223.45.101`
- **SSH用户**: `ubuntu` 或 `root`
- **后端部署路径**: `/opt/website-batch-system/backend/`
- **前端部署路径**: `/opt/website-batch-system/`（根目录）
- **后端服务名**: `website-batch-system`
- **访问端口**: `8088`（Nginx代理）

---

## 🚀 快速部署（推荐）

### 方法一：使用部署脚本

在项目根目录执行：

```bash
# 给脚本执行权限
chmod +x quick-deploy.sh

# 执行部署（会自动构建前端和后端）
bash quick-deploy.sh 124.223.45.101
```

### 方法二：使用完整部署脚本

```bash
chmod +x deploy-full.sh
bash deploy-full.sh 124.223.45.101
```

然后按照提示在服务器上执行相关命令。

## 📝 手动部署步骤

如果自动脚本无法使用，请按以下步骤手动部署：

### 第一步：本地构建

#### 1. 构建后端

```bash
cd backend
mvn clean package -DskipTests
```

构建完成后，JAR文件位于：`backend/target/website-batch-system-1.0.0.jar`

#### 2. 构建前端

```bash
cd frontend
npm install
npm run build
```

构建完成后，静态文件位于：`frontend/dist/`

---

### 第二步：上传文件到服务器

#### 1. 上传后端JAR

```bash
# 方法1：直接上传
scp backend/target/website-batch-system-1.0.0.jar ubuntu@124.223.45.101:/tmp/

# 方法2：使用root用户
scp backend/target/website-batch-system-1.0.0.jar root@124.223.45.101:/tmp/
```

#### 2. 上传前端文件

```bash
# 打包前端文件
tar -czf frontend-dist.tar.gz -C frontend/dist .

# 上传到服务器
scp frontend-dist.tar.gz ubuntu@124.223.45.101:/tmp/
```

---

### 第三步：SSH登录服务器

```bash
# 使用ubuntu用户
ssh ubuntu@124.223.45.101

# 或使用root用户
ssh root@124.223.45.101
```

---

### 第四步：服务器上部署

#### 1. 停止后端服务

```bash
sudo systemctl stop website-batch-system
```

#### 2. 备份旧版本（可选但推荐）

```bash
# 创建备份目录
timestamp=$(date +%Y%m%d_%H%M%S)
sudo mkdir -p /opt/website-batch-system/backups/$timestamp

# 备份后端JAR
if [ -f /opt/website-batch-system/backend/website-batch-system-1.0.0.jar ]; then
    sudo cp /opt/website-batch-system/backend/website-batch-system-1.0.0.jar \
        /opt/website-batch-system/backups/$timestamp/
fi

# 备份前端文件
sudo cp -r /opt/website-batch-system/assets /opt/website-batch-system/backups/$timestamp/ 2>/dev/null || true
sudo cp /opt/website-batch-system/index.html /opt/website-batch-system/backups/$timestamp/ 2>/dev/null || true

echo "备份完成: /opt/website-batch-system/backups/$timestamp/"
```

#### 3. 部署后端

```bash
# 替换JAR文件
sudo cp /tmp/website-batch-system-1.0.0.jar /opt/website-batch-system/backend/

# 设置权限
sudo chmod 755 /opt/website-batch-system/backend/website-batch-system-1.0.0.jar
```

#### 4. 部署前端

```bash
# 清理旧前端文件
sudo rm -rf /opt/website-batch-system/assets
sudo rm -f /opt/website-batch-system/index.html

# 解压新前端文件
sudo tar -xzf /tmp/frontend-dist.tar.gz -C /opt/website-batch-system/

# 设置权限（根据Web服务器用户调整）
sudo chown -R www-data:www-data /opt/website-batch-system/assets 2>/dev/null || \
sudo chown -R nginx:nginx /opt/website-batch-system/assets 2>/dev/null || true
sudo chown www-data:www-data /opt/website-batch-system/index.html 2>/dev/null || \
sudo chown nginx:nginx /opt/website-batch-system/index.html 2>/dev/null || true
```

#### 5. 清理临时文件

```bash
rm -f /tmp/website-batch-system-1.0.0.jar
rm -f /tmp/frontend-dist.tar.gz
```

---

### 第五步：重启服务

#### 1. 启动后端服务

```bash
sudo systemctl start website-batch-system
```

#### 2. 重启Nginx

```bash
sudo systemctl restart nginx
```

---

### 第六步：验证部署

#### 1. 检查后端服务状态

```bash
sudo systemctl status website-batch-system
```

应该显示 `active (running)` 状态。

#### 2. 检查端口监听

```bash
sudo netstat -tulpn | grep -E ':8080|:80'
```

应该看到：
- `8080` - 后端服务
- `80` - Nginx

#### 3. 测试API

```bash
# 测试后端健康状态
curl http://localhost:8080/api/excel/companies

# 测试前端
curl -I http://localhost/
```

#### 4. 查看日志（如果有问题）

```bash
# 后端日志
sudo journalctl -u website-batch-system -f

# 最近100行日志
sudo journalctl -u website-batch-system -n 100

# Nginx错误日志
sudo tail -f /var/log/nginx/error.log
```

## 🔄 回滚操作

如果部署出现问题，可以快速回滚：

```bash
# 停止服务
sudo systemctl stop website-batch-system

# 恢复旧版本JAR
timestamp="备份时的时间戳"  # 例如：20260331_120000
sudo cp /opt/website-batch-system/backups/$timestamp/website-batch-system-1.0.0.jar \
    /opt/website-batch-system/backend/

# 恢复前端文件
sudo cp -r /opt/website-batch-system/backups/$timestamp/assets /opt/website-batch-system/
sudo cp /opt/website-batch-system/backups/$timestamp/index.html /opt/website-batch-system/

# 启动服务
sudo systemctl start website-batch-system
sudo systemctl restart nginx
```

## ⚠️ 注意事项

1. **部署前先测试**: 确保本地构建没有错误
2. **备份旧版本**: 部署前备份可以快速回滚
3. **检查服务状态**: 部署后务必验证服务是否正常运行
4. **查看日志**: 如果有问题，第一时间查看日志

## 📞 故障排查

### 后端无法启动

```bash
# 查看详细错误
sudo journalctl -u website-batch-system -n 200

# 常见原因：
# 1. 数据库连接失败 - 检查数据库配置
# 2. 端口被占用 - 检查8080端口
# 3. 内存不足 - 检查服务器内存
```

### 前端无法访问

```bash
# 检查Nginx状态
sudo systemctl status nginx

# 检查文件权限
ls -la /opt/website-batch-system/

# 检查Nginx配置
sudo nginx -t
```

### API返回错误

```bash
# 检查后端日志
sudo journalctl -u website-batch-system -n 100

# 测试后端直接访问
curl http://localhost:8080/api/excel/companies
```
